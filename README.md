# ระบบบันทึกกิจกรรมนักศึกษา (EDU Activity Record)
### คณะครุศาสตร์ มหาวิทยาลัยราชภัฏกำแพงเพชร

ระบบบันทึกและจัดการการเข้าร่วมกิจกรรมของนักศึกษา พัฒนาในรูปแบบ Web Application ที่รันบนฝั่ง Client 100% (ไม่มีฐานข้อมูลภายนอกและไม่มีเซิร์ฟเวอร์หลังบ้าน) ทำให้สามารถเข้าใช้งานกล้องเพื่อสแกน QR Code / Barcode จากบัตรนักศึกษา และทำงานแบบออฟไลน์ได้อย่างเสถียร พร้อมทั้งสามารถส่งออกรายงานและรวมข้อมูลจากเครื่องสแกนอื่นๆ ได้อย่างง่ายดาย

---

## 🌟 คุณสมบัติเด่นของระบบ (Key Features)

1. **ระบบลงบันทึกด้วย QR / Barcode**:
   - เปิดสแกนรหัสผ่านกล้องของสมาร์ทโฟน แท็บเล็ต หรือคอมพิวเตอร์ได้ทันที
   - ตรวจสอบรูปแบบรหัสนักศึกษาโดยอัตโนมัติ (ตัวเลข 9 หลัก เช่น `661121301`)
   - มีระบบป้องกันการสแกนซ้ำซ้อน (Duplicate registration prevention) สำหรับแต่ละกิจกรรม
   - สัญญาณเสียงบี๊บ (Web Audio API) และการสั่นแจ้งเตือนเมื่อการสแกนผ่านหรือมีข้อผิดพลาด
2. **ระบบจัดการกิจกรรม**:
   - สามารถกำหนดกิจกรรมใหม่พร้อมวันที่และเวลาเริ่มต้นของกิจกรรมได้ในตัวระบบ
   - แสดงรายการกิจกรรมทั้งหมดพร้อมระบุจำนวนนักศึกษาที่ลงทะเบียนสำเร็จในกิจกรรมนั้นๆ
3. **แดชบอร์ดสรุปผลและส่งออกรายงาน**:
   - แสดงสถิติรวมของนักศึกษา สรุปจำนวนผู้เข้าร่วมแบบแยกกิจกรรมเดี่ยวๆ หรือดูรวมทุกกิจกรรม
   - ตารางประวัติการบันทึกสามารถค้นหาด้วยรหัสนักศึกษาได้รวดเร็ว
   - ดาวน์โหลดรายงานประวัติการเข้าร่วมกิจกรรมเป็นไฟล์ **Excel (.xlsx)** หรือ **CSV** (รองรับภาษาไทยใน Microsoft Excel ได้อย่างถูกต้องสมบูรณ์)
4. **ระบบรวมข้อมูลสำรอง (Multi-Device Backup Sync)**:
   - ออกแบบมาเพื่อแก้ปัญหาการลงทะเบียนที่มีหลายช่องทางและไม่มีเซิร์ฟเวอร์กลาง
   - สามารถส่งออกข้อมูลสำรอง (JSON Backup) จากเครื่องย่อยๆ นำมาอัปโหลดรวมร่าง (Merge & Sync) กันบนคอมพิวเตอร์หลักเพื่อดาวน์โหลดรายงานแดชบอร์ดรวมได้ทันที
5. **ความสวยงามและใช้งานง่าย (UX/UI)**:
   - ดีไซน์สไตล์ Glassmorphism สวยงามล้ำสมัย
   - มีหน้ากล้องที่พร้อม HUD กรอบเป้าเล็งและเส้นเลเซอร์สแกนเสมือนจริง
   - รองรับธีมมืดและธีมสว่าง (Dark / Light Mode Toggle)
   - ปรับการจัดวางเหมาะสมกับทุกอุปกรณ์ (Responsive design)

## 🚀 ขั้นตอนการเปิดใช้งานผ่าน GitHub Pages

ระบบนี้รองรับการเผยแพร่ผ่าน GitHub Pages ได้อย่างสมบูรณ์แบบ เนื่องจากไม่มีระบบ Backend ที่ซับซ้อน สามารถทำตามวิธีนี้ได้:

### 1. นำโค้ดขึ้น GitHub Repository
ในคอมพิวเตอร์ของคุณที่โฟลเดอร์โครงการนี้ ให้รันคำสั่งเหล่านี้ผ่าน Git Bash หรือ Command Prompt เพื่อ Push ขึ้น Repository:
```bash
# ตรวจสอบการเพิ่มไฟล์
git add .

# บันทึกเวอร์ชันโค้ด
git commit -m "Initial commit: EDU Activity Record System"

# สร้าง Branch หลัก
git branch -M main

# อัปโหลดขึ้นไปยัง GitHub (ที่ https://github.com/tswiwat/eduAct.git)
git push -u origin main
```
*(หมายเหตุ: ในครั้งแรกระบบจะให้คุณลงชื่อเข้าใช้งานสิทธิ์บัญชี GitHub ของคุณก่อนอัปโหลด)*

### 2. ตั้งค่าเปิดใช้ GitHub Pages บนเว็บ GitHub
1. เปิดบราวเซอร์ไปที่หน้าคลังโค้ดของคุณบน GitHub: `https://github.com/tswiwat/eduAct`
2. ไปที่เมนู **Settings** (รูปไอคอนฟันเฟืองด้านบนสุดของขวาหน้าต่าง)
3. เมนูด้านซ้าย มองหาและคลิกที่หัวข้อ **Pages**
4. ในส่วนของ **Build and deployment**:
   - ใต้หัวข้อ *Source* เลือก **Deploy from a branch**
   - ใต้หัวข้อ *Branch* เลือก **main** (หรือ `master`) จากนั้นเปลี่ยนไดเรกทอรีถัดไปเป็น `/ (root)`
5. กดปุ่ม **Save** สีน้ำเงิน
6. รอประมาณ 1-2 นาที จากนั้นให้รีเฟรชหน้าเว็บนั้นขึ้นมาใหม่ จะปรากฏลิงก์สีเขียวระบุว่า:
   > **Your site is live at `https://tswiwat.github.io/eduAct/`**
7. คลิกที่ลิงก์ดังกล่าวเพื่อเปิดใช้งานระบบสแกนบัตรและลงทะเบียนบันทึกกิจกรรมผ่านอุปกรณ์มือถือ/แท็บเล็ตของคุณได้ทันที!

## 🛠️ คำแนะนำและการแก้ไขปัญหา

* **ปัญหาไม่พบคอมพิวเตอร์ยื่นสิทธิ์เปิดกล้อง**:
  - มือถือ iOS และ Android จะบังคับให้เว็บแอปเรียกสิทธิ์ใช้กล้องได้เฉพาะเว็บที่ทำงานบน **HTTPS (ปลอดภัย)** เท่านั้น การรันผ่านระบบจำลองโลคอล (http://localhost) หรือการรันจาก GitHub Pages (ซึ่งเป็น https) จะเรียกใช้งานกล้องได้อย่างปลอดภัยไร้ปัญหา
  - ตรวจสอบว่าแอปเบราว์เซอร์ของคุณ (เช่น Safari, Chrome) ได้รับสิทธิ์อนุญาตใช้กล้องถ่ายรูปในการตั้งค่าระบบของตัวเครื่องแล้วหรือไม่
* **ข้อมูลสูญหาย**:
  - เนื่องจากระบบเก็บข้อมูลไว้ในหน่วยความจำของเบราว์เซอร์ (`localStorage`) ข้อมูลจะไม่หายไปหากปิดเครื่องหรือรีเฟรชหน้าเว็บ 
  - **ห้าม** กดล้างคุ้กกี้ ล้างประวัติการเข้าชมเว็บ หรือใช้หน้าต่างไม่ระบุตัวตน (Incognito Mode) ในการทำงานหลัก เพราะอาจส่งผลให้เบราว์เซอร์ทำการเคลียร์ล้างข้อมูล `localStorage` ทิ้ง
  - **แนะนำ:** ให้ใช้ปุ่ม **ส่งออกข้อมูลสำรอง (JSON Backup)** ในหน้าการตั้งค่าทุกครั้งหลังสแกนกิจกรรมเสร็จสิ้น เพื่อเป็นการป้องกันข้อมูลสูญหายและมีประวัติสำรองไว้อีกระดับหนึ่ง
